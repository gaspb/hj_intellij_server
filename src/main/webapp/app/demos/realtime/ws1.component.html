This module is currently powered by :
-Spring WebSocket and SockJS for sending and receiving instantly messages
-Hazelcast, as a distributed cache throughout the application, let us keep in memory the last X messages(here just 20 for easy testing) for each topic in an evicting queue, and serve them instantly to connecting clients. We didn't even use a database for this demo. We will though add a Cassandra instance connected to Kafka to complement HazelCast.
This module will then be added to the Gateway.
-Angular5 as the renderer. Even if it was almost perfect for the job, we will try soon to implement a second front end on React !
Another completely standalone version will also be developped, using Kafka Zookeeper and Cassandra, without the need of a backend.

Any ideas or request features ? Join our Jira :)
